# Loader 与 Plugin 的区别

  • Loader：在模块解析阶段处理资源文件，将非 JS 资源转换为可打包的模块。例如：babel-loader、css-loader、file-loader。
  • Plugin：在构建生命周期的各个阶段插入逻辑，扩展打包能力。例如：HtmlWebpackPlugin、DefinePlugin、MiniCssExtractPlugin。

# 常见 Loader 示例

module.exports = {
  module: {
    rules: [
      { test: /\.jsx?$/, exclude: /node_modules/, use: 'babel-loader' },
      { test: /\.css$/, use: ['style-loader', 'css-loader'] },
      { test: /\.(png|jpg|gif|svg)$/i, type: 'asset' }
    ]
  }
}

# 常见 Plugin 示例

const HtmlWebpackPlugin = require('html-webpack-plugin');
const { DefinePlugin } = require('webpack');
module.exports = {
  plugins: [
    new HtmlWebpackPlugin({ template: './public/index.html' }),
    new DefinePlugin({ __APP_VERSION__: JSON.stringify('1.0.0') })
  ]
}

# 选择与调优建议

  • 优先使用内置 asset 模式处理静态资源，减少额外 loader。
  • Plugin 过多会增加构建时间，按需启用并注意顺序与版本兼容。
  • 对于大型项目，结合 cache、thread-loader 与 babel 的 cacheDirectory 提升性能。
